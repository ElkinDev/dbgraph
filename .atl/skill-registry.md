## Compact Rules

### dbgraph-conventions — when touching `src/**`

- Hexagonal boundaries (ADR-004), NON-NEGOTIABLE: `src/core` imports NOTHING from
  `src/adapters`/`src/mcp`/`src/cli` nor any DB driver; adapters import only core types/ports +
  their own driver via dynamic `import()`; mcp/cli import only the public core API.
- Naming: kebab-case files, PascalCase types, camelCase functions. Ports named by role
  (`SchemaAdapter`, `GraphStore`, `Logger`); implementations prefixed by dialect (`MssqlSchemaAdapter`).
- Errors: typed classes (`ConnectionError`, `PermissionError` with exact missing permission,
  `ConfigError`, `UnsupportedDialectError`); actionable messages; never swallow; never throw strings.
- Strict TS: no `any` (use `unknown` + narrowing); no `console.log` in core (use `Logger` port);
  public API only via `src/index.ts` / `src/core/index.ts`.

### dbgraph-testing — when writing tests or testable code

- STRICT TDD (red→green→refactor) for ALL of `src/core` and every output format: the failing
  test exists BEFORE the implementation.
- Golden files pin every MCP/CLI output format; changing a golden requires stating the intent;
  goldens are NECESSARY but NOT SUFFICIENT — add targeted endpoint assertions for critical edges.
- Adapters: integration-first against the engine's torture schema (Testcontainers; SQLite uses a
  committed fixture). NEVER mock the driver in integration tests.
- Edge/graph tests MUST assert endpoints (both src AND dst qnames), never mere existence. An
  existence-only assertion let a phantom-stub trigger target (C-1) pass green CI. For every
  semantically critical edge kind (fires_on, writes_to, reads_from, references): assert src qname,
  dst qname, and that no wrong-target/stub variant exists at the dst (L-009).
- NEVER drop a spec-mandated assertion because a platform limitation is SUSPECTED — always verify
  empirically against the live system first; undocumented removal = CRITICAL in verify (C-2).
- Fixtures at `test/fixtures/catalog-*.json`; AAA structure; one behavior per test; story
  acceptance criteria (US-xxx) map to at least one test each.

### dbgraph-security — when touching adapters/connection/config/dependencies

- Read-only by construction: adapters emit ONLY catalog SELECTs; a repo test scans embedded SQL
  for write verbs and fails on a hit; integration runs use a user that CANNOT write.
- Secrets: `${env:VAR}` references only; config writer REJECTS plaintext credentials; never log
  a resolved URL; `.dbgraph/` always gitignored.
- MongoDB sampling persists keys+types only, never values (tested).
- Dependencies: closed canonical list (`mssql`, `pg`, `mysql2`, `mongodb`, `better-sqlite3`,
  `@modelcontextprotocol/sdk` + dev toolchain); official registry only; new dependency = written
  justification in the PR.
