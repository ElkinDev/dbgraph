- **Batch B** (2.1–2.8): `openConnections` move + barrel re-export + CLI import update + boundary test, pinned SDK, **SDK API verify**, instructions golden, server scaffold, harness skeleton + initialize golden.

- **Batch D** (4.1–4.5): `object` + `impact` orchestrators, precheck (PURE extractor unit → match → aggregate), `affected` CLI sibling over the shared core.

### Dependency bottlenecks

- 1.1 (format-spec grammar) gates 1.2–1.8 (formatters render to that grammar); 1.9 (barrel) gates ALL tool tasks that import formatters.
- 2.1+2.2 (`openConnections` relocation + barrel re-export) gate every tool (the harness + server open connections via the barrel) — must land + CLI stay green FIRST.
- 2.3 (boundary test) + 2.5 (SDK API verify) gate 2.7 (server) — do NOT wire tools before the SDK shape is confirmed against the pinned version.
- 2.8 (harness) gates ALL tool goldens in Batches C/D/E.
- 4.3 (`extractIdentifiers` unit) gates 4.4 (`precheck` tool) and 4.5 (`affected`); the shared core must NOT live under `src/mcp/**` or `affected` violates the CLI boundary.
- 5.4 (budget pin) depends on 5.3 (E2E goldens exist to measure) — budgets stay **TBD** until the E2E output is measured.
