module is PURE core (ADR-004): `src/core/infer/` imports ONLY core model types (`node.ts`/`edge.ts`/`capability.ts`)

Strict TS, no `any`, `exactOptionalPropertyTypes`. dbgraph-security: read ONLY node `name`/`qname`/`payload.dataType`
‚Äî NEVER a raw data value. Endpoint grain is COLUMN‚ÜíCOLUMN (D1): an emitted edge's `src`/`dst` are the two column/field
node ids `nodeId('column', srcQName)` / `nodeId('column', targetColQName)` (NOT table‚Üítable); `attrs.srcColumn` =
matched local name, `attrs.dstColumn` = resolved target column local name. PK-of-target is read from `constraint`
child nodes (`payload.type==='PK'`, `payload.columns` case-folded) ‚Äî `ColumnPayload` has NO `isPrimaryKey` (D2).
Score = `W_CONVENTION*conv + W_TYPE*typeCompat + W_PK_TARGET*targetIsPk` with NAMED constants
`W_CONVENTION=0.5, W_TYPE=0.3, W_PK_TARGET=0.2, THRESHOLD=0.5`; `conv‚àà{1.0 for <e>_id/<e>Id, 0.8 for id_<e>}`,
`typeCompat‚àà{1,0}`, `targetIsPk‚àà{1,0}`; emit ONLY when `score >= THRESHOLD` (below ‚Üí no edge). Type incompatibility
is a HARD REJECT in the matcher BEFORE scoring ‚Äî a mismatch still scores `0.5+0.2=0.7 ‚â• THRESHOLD`, so penalizing is
NOT enough; reject before scoring so NO edge is emitted. Skip a column that already carries a declared `references`
edge (existingEdges dedup ‚Äî no inferred/declared overlap). The engine SELF-SORTS its output `(src, dst, score DESC,
attrs.srcColumn, id)` BEFORE returning, because `normalize.ts` `sortEdges` (L506‚Äì519) has NO `score` key (D4/ADR-008).
EXACT-set assertions ALWAYS (L-009): `toEqual`/`toBe` the FULL emitted edge (exact src qname + dst qname + score +
`attrs.srcColumn`/`attrs.dstColumn`), `toBe(N)` counts, explicit `not.toContainEqual` for negative cases ‚Äî existence-only
`find().toBeDefined()` is FORBIDDEN. Conventional commits referencing US-008; ENGLISH.

### Dependency bottlenecks

- **Batch 1 gates Batch 2.** `compatible(a,b)` (the hard-reject) and `extractEntity`/`candidateTargets` are consumed by the engine's match loop. Their EXACT-set tests must be green before the engine wires them ‚Äî otherwise an over-eager pluralization or a wrong family bug surfaces as a confusing engine failure instead of a precise unit failure.
- **Batch 2 gates Batch 3.** The hook calls `inferReferences(nodeMap, edges, {})`; the engine's self-sort + COLUMN‚ÜíCOLUMN grain + score must be proven (incl. the negative `status_id` and type-incompat cases) BEFORE the normalizer wires it, so a gate-ON regression is isolated to the hook, not the algorithm.
- **3.4 (gate-OFF byte-identical) is the load-bearing check of the whole change.** It depends on 3.1 (the optional field) + 3.2 (the gated hook). A golden drift here means the hook ran on the OFF path (the `scope.inferRelationships === true` / `collection`/`field` guard is wrong) ‚Äî a hard STOP, NOT a re-bless. With NO CI, this local gate is the ONLY thing standing between this change and a silent break of the four shipped SQL engines' goldens.
- **The hook MUST self-sort via the engine (D4), NOT lean on `sortEdges`.** `sortEdges` (L506‚Äì519) has no `score` key, so multi-candidate order would be input-dependent if the engine did not pre-sort. Splitting determinism between the engine and the comparator would re-litigate the resolved D4 decision ‚Äî do NOT.
